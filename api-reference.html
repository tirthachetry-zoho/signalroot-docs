SignalRoot API Reference

Base URL

Production: https://api.signalroot.com
Development: http://localhost:8080

Authentication

SignalRoot uses API key authentication for secure access.

Getting Your API Key

  1. Navigate to Settings ‚Üí API Keys in the dashboard
  2. Click Generate New Key
  3. Copy the key and store it securely

Using the API Key

Include the API key in the Authorization header:

curl -H "Authorization: Bearer YOUR_API_KEY" \
  https://api.signalroot.com/api/teams

Core Endpoints

Health Check

GET /actuator/health

Check the health status of the SignalRoot service.

Response:

{
  "status": "UP",
  "components": {
    "db": {
      "status": "UP",
      "details": {
        "database": "PostgreSQL",
        "validationQuery": "isValid()"
      }
    }
  }
}

Teams Management

Get All Teams

GET /api/fake-customers/teams
Authorization: Bearer YOUR_API_KEY

Retrieve all configured teams with their services and alert frequencies.

Get Team Details

GET /api/fake-customers/teams/{teamId}
Authorization: Bearer YOUR_API_KEY

Get detailed information about a specific team.

Get Team Statistics

GET /api/fake-customers/teams/{teamId}/stats
Authorization: Bearer YOUR_API_KEY

Retrieve statistics and metrics for a specific team.

Alert Processing

Process PagerDuty Alert

POST /webhooks/alerts/pagerduty
Content-Type: application/json

Process PagerDuty alerts with automatic enrichment.

Request Body:

{
  "messages": [{
    "type": "incident",
    "incident": {
      "id": "INC123",
      "status": "triggered",
      "title": "API Gateway High Latency",
      "service": "payment-service",
      "severity": "high",
      "details": {
        "impacted_services": ["payment-api", "checkout-flow"],
        "error_rate": "12%",
        "response_time": "2.3s"
      }
    }
  }]
}

Dogfooding Scenarios

Get All Scenarios

GET /api/dogfooding/scenarios
Authorization: Bearer YOUR_API_KEY

Retrieve all available dogfooding scenarios for testing.

Run All Scenarios

POST /api/dogfooding/run-all
Authorization: Bearer YOUR_API_KEY

Execute all dogfooding scenarios to test the system.

Run Specific Scenario

POST /api/dogfooding/scenarios/{scenarioId}/run
Authorization: Bearer YOUR_API_KEY

Execute a specific dogfooding scenario.

Version Management

Get Current Version

GET /api/versions/version/current
Authorization: Bearer YOUR_API_KEY

Get the current version and release information.

Get Changelog

GET /api/versions/changelog/{version}
Authorization: Bearer YOUR_API_KEY

Retrieve the changelog for a specific version.

JavaScript Example

import { SignalRootClient } from '@signalroot/sdk';

const client = new SignalRootClient({
  apiKey: 'YOUR_API_KEY',
  baseUrl: 'https://api.signalroot.com'
});

// Get teams
const teams = await client.getTeams();

// Process alert
const result = await client.processAlert({
  type: 'incident',
  service: 'payment-service',
  severity: 'high',
  title: 'API latency spike'
});

Rate Limits

Plan Requests/Minute Requests/Hour Requests/Day
Free 60 1,000 10,000
Pro 600 10,000 100,000
Enterprise 6,000 100,000 1,000,000

Rate limit headers are included in responses:

X-RateLimit-Limit: 60
X-RateLimit-Remaining: 45
X-RateLimit-Reset: 1642694400

Error Handling

Standard Error Response

{
  "error": {
    "code": "INVALID_API_KEY",
    "message": "The provided API key is invalid or expired",
    "details": {
      "timestamp": "2024-01-20T16:51:04.533015Z",
      "requestId": "req_abc123"
    }
  }
}
Code Description Solution
INVALID_API_KEY API key is invalid or expired Generate a new API key
RATE_LIMIT_EXCEEDED Too many requests Wait and retry
TEAM_NOT_FOUND Team ID doesn't exist Check team ID
